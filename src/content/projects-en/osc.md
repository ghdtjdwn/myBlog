## Current scope

This is a personal workspace for organizing the official rules, submission format, and supplied templates for an open-source developer competition. No target project, GitHub Issue, code change, Pull Request, or maintainer review exists yet, so it is classified as preparation rather than a technical project.

## Publication policy

At present, there are submission documents but no code or open-source contribution. The contribution process and lessons learned will be documented only after a real Issue, Pull Request, and review exist.

Once work begins, the record will connect the evidence used to identify a problem, the Issue that establishes scope with a maintainer, the implementation commit, decisions changed during review, and whether the work reached a release. Even if the PR is not merged, verifiable feedback and iteration may support a retrospective; preparation documents will never be presented as an open-source contribution.
