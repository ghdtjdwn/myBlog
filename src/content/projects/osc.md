## 현재 범위

오픈소스 개발자대회 참가를 위해 원본 규정, 제출 형식과 제공 템플릿을 정리한 개인 작업공간입니다. 아직 대상 프로젝트 선정, GitHub Issue, 코드 변경, Pull Request나 maintainer review가 없으므로 기술 프로젝트가 아니라 준비 단계로 분류합니다.

## 공개 원칙

현재는 제출 문서만 있고 코드나 오픈소스 기여가 없습니다. 실제 issue, pull request, review가 생긴 뒤 기여 과정과 배운 점을 기록합니다.

실제 기여가 시작되면 문제를 발견한 근거, maintainer와 범위를 합의한 Issue, 구현 commit, review에서 바뀐 판단과 release 포함 여부를 하나의 흐름으로 남깁니다. PR이 merge되지 않더라도 피드백과 개선 과정이 확인 가능하면 회고할 수 있지만, 준비 문서를 오픈소스 기여처럼 표현하지 않습니다.

## 다음 단계

관심 프로젝트의 contribution guide와 open issue를 확인하고, 작은 문서·테스트 개선부터 maintainer에게 범위를 제안합니다. 실제 공개 링크가 생긴 뒤 상태를 `planned`에서 변경하고 프로젝트 페이지에 원문을 연결합니다.
