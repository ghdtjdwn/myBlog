## 핵심 결정

교육 콘텐츠에서 AI가 설명과 대사를 자유 생성하면 틀린 내용을 가르칠 위험이 있습니다. 문제·정답·대사는 사람이 작성하고 AI는 정답 anchor와 학습자의 답을 비교해 판정만 반환하게 했습니다.

학습자가 자유서술로 답하더라도 모델이 게임 규칙을 바꾸지는 못합니다. grader는 제한된 verdict 계약만 반환하고, 호감도·문제 순서·재시도·엔딩은 `engine.js`가 소유합니다. 모델 응답이 없거나 형식이 틀려도 상태 전이가 망가지지 않도록 책임을 나눴습니다.

## 장애 격리

호감도, 문제 순서, 재설명, 엔딩은 결정론 엔진이 소유합니다. 서버나 AI 키가 없는 정적 배포에서는 mock grader로 완주할 수 있고, 실제 AI 채점은 서버 프록시를 사용할 때만 활성화됩니다.

정적 fallback은 단순 시연용 가짜 화면이 아니라 실제 grader와 같은 계약을 구현합니다. 브라우저에 API key를 넣지 않고, 서버 프록시가 있을 때만 OpenAI 또는 Claude 채점을 선택합니다. 따라서 외부 AI가 실패해도 사람이 작성한 콘텐츠와 게임 완주 경로는 남습니다.

## 구현과 검증

현재 공개 가능한 기록으로는 23개 engine test, `stack.json`의 8개 beat 한 회차, 오답 후 재설명과 재시도, 호감도·엔딩 불변식과 Playwright 완주가 있습니다. 이 수치는 전체 교육 콘텐츠의 학습효과가 아니라 게임 엔진의 완주 가능성을 확인한 것입니다.

## 현재 상태

스택 자료구조 루트와 자체 UI·아트 데모는 완료했습니다. 큐·트리 루트와 카메오 심화 콘텐츠, 실제 grader의 경계 사례 튜닝, 공개 배포는 아직 작업 중입니다. 제 역할은 스토리·정답 anchor, UI·대화창과 아트 설계 범위이며 엔진과 서버 전체를 개인 구현으로 표현하지 않습니다. 따라서 전체 제품을 완료 상태로 표시하지 않고 프로토타입으로 공개 범위를 제한합니다.

## 기여 경계

이 프로젝트는 팀 작업입니다. 블로그에서는 확인된 개인 역할과 설계 과정만 설명하고 팀 전체 결과를 단독 구현으로 표현하지 않습니다.

저장소가 비공개이고 과거 이미지 자산의 공개 권리도 정리가 필요해 현재 스크린샷은 사용하지 않습니다. 실제 학습효과와 사용자 지표도 측정하지 않았으므로 주장하지 않습니다.
