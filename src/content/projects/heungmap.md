## 해결하려는 문제

축제의 흥행 가능성을 주관적인 기대가 아니라 지역·시기·방문자 데이터로 설명하고, 방문객에게는 관심도와 혼잡 가능성을 함께 보여주는 관광데이터 공모전 아이디어입니다. 2인 팀으로 문제와 후보 데이터를 정리했지만 세부 구현 역할은 아직 확정하지 않았습니다.

## 지금 확인할 것

모델부터 만드는 대신 축제 단위 라벨과 지역·시기·방문자 데이터가 실제로 조인되는지 확인합니다. 데이터 누수, 관측 기간, 예측 시점에 사용할 수 없는 변수를 먼저 제거해야 예측 결과를 제품 기능으로 설명할 수 있습니다.

계획은 TourAPI의 축제 메타데이터와 지역·기간 방문자 데이터를 연결해 50~100개 축제의 최소 데이터셋을 만드는 것입니다. 먼저 규칙 기반 기준선을 만들고, 라벨 품질이 확인된 뒤에만 ML과 설명 가능한 feature를 검토합니다. 모델 정확도를 먼저 높이는 대신 “축제 단위 수요”라고 부를 수 있는 라벨인지부터 판정합니다.

## 현재 상태와 다음 기준

현재는 참가 신청과 기획 문서까지 완료했으며 API key 발급, 실데이터 수집, label join 성공률은 아직 확인하지 않았습니다. 데이터 스파이크에서 지역 단위 방문자를 개별 축제에 귀속할 수 없으면 혼잡 예측이나 탐색 문제로 피벗합니다. 이 go/no-go 결과 자체를 작업 로그로 남길 예정입니다.

## 공개 원칙

현재는 기획과 참가 신청 단계입니다. 모델 정확도나 배포 결과를 주장하지 않고, 데이터 스파이크에서 확인한 사실과 기술 선택만 기록합니다.

공모전 결과와 무관하게 문제 정의, 사용할 수 없는 변수와 실패한 조인 가설을 공개 가능한 범위에서 기록합니다. 실제 구현 전에 계획 스택을 사용 경험처럼 나열하지 않습니다.
